# Contributing to GrowIN

Thank you for your interest in contributing to GrowIN.

## Getting Started

1. Fork the repository.
2. Create a new feature branch.
3. Make your changes.
4. Commit using clear commit messages.
5. Open a Pull Request.

## Development Guidelines

* Follow the existing project structure.
* Keep components modular and reusable.
* Write clean, documented code.
* Avoid unnecessary dependencies.
* Test your changes before submitting.

## Branch Naming

* `feature/...`
* `fix/...`
* `docs/...`
* `refactor/...`

## Commit Style

Examples:

* feat: add AI interview flow
* fix: resolve authentication issue
* docs: update README
* refactor: simplify prompt engine

## Code Style

* Use meaningful variable names.
* Keep functions focused on one responsibility.
* Follow Python and TypeScript best practices.
* Add comments only where necessary.

## Reporting Issues

When creating an issue, please include:

* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots (if applicable)
* Environment details

Thank you for helping improve GrowIN.
